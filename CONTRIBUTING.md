# Contributing to IPv6 AI Exposure Research

Thank you for your interest in contributing!

## How to Contribute

### Reporting Issues
- Use GitHub Issues with the bug/feature template
- Include environment details and reproduction steps
- For security vulnerabilities, use private disclosure

### Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make changes with clear commit messages
4. Add tests for new functionality
5. Ensure code passes linting (`flake8`, `pylint`)
6. Submit PR with description of changes

### Coding Standards
- Follow PEP 8 for Python code
- Add docstrings to all functions
- Include type hints where applicable
- Write unit tests for new features

### Detection Rule Contributions
- Follow Snort/Suricata rule syntax
- Include SID in the 9000000+ range
- Document the detection logic
- Test against sample traffic

## Research Collaboration

For research collaborations or data sharing, please contact the research team.
